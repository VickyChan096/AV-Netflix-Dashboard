# 如何更新片單

網站的所有內容都來自這一個檔案：

```
src/data/watchlist.json
```

你每次看完一部片，就到公司打開這個檔案、加一筆資料，然後 push 到 GitHub，
GitHub 會自動重新部署，網站幾分鐘後就會更新。

---

## 一筆資料長這樣

```json
{
  "title": "愛在黎明破曉時",
  "type": "movie",
  "status": "watched",
  "genre": "愛情",
  "cover": "https://圖片網址.jpg",
  "ratingVicky": 5,
  "ratingAllen": 4,
  "watchedOn": "2026-07-12",
  "note": "兩個人的心得"
}
```

整個檔案是一個「清單」，用中括號 `[ ]` 包住，每一筆用大括號 `{ }`，
**每一筆之間要用逗號 `,` 隔開，最後一筆後面不要加逗號。**

---

## 各欄位說明

| 欄位 | 必填 | 說明 |
|------|:----:|------|
| `title` | ✅ | 片名 |
| `type` | ✅ | `"movie"`（電影）或 `"series"`（劇集） |
| `status` | ✅ | `"watched"`（已看過）或 `"wishlist"`（想看） |
| `genre` | | 分類，例如 `"愛情"`、`"科幻"`、`"喜劇"` |
| `cover` | | 封面圖網址；留空 `""` 會顯示預設圖示 |
| `ratingVicky` | | Vicky 評分，0～5 的數字 |
| `ratingAllen` | | Allen 評分，0～5 的數字 |
| `watchedOn` | | **電影**用：一起看的日期，格式 `"2026-07-12"` |
| `startedOn` | | **劇集**用：開始看的日期 |
| `finishedOn` | | **劇集**用：看完的日期（統計會算在這個月） |
| `note` | | 心得，一句話也行 |

> 不需要的欄位可以整行刪掉，不用留空。

---

## 兩個常見範例

**看完一部電影：**

```json
{
  "title": "可憐的東西",
  "type": "movie",
  "status": "watched",
  "genre": "劇情",
  "ratingVicky": 4,
  "ratingAllen": 5,
  "watchedOn": "2026-07-20",
  "note": "很怪但很喜歡"
}
```

**追完一部劇（跨月）：**

```json
{
  "title": "柏捷頓家族 第三季",
  "type": "series",
  "status": "watched",
  "genre": "愛情",
  "ratingVicky": 5,
  "ratingAllen": 4,
  "startedOn": "2026-06-20",
  "finishedOn": "2026-07-08"
}
```

**還沒看、先加進想看清單：**

```json
{
  "title": "沙丘 2",
  "type": "movie",
  "status": "wishlist",
  "genre": "科幻"
}
```

---

## 小提醒

- 所有文字、日期都要用**雙引號** `"..."` 包起來；只有分數（數字）不用。
- 日期一律用 `"年-月-日"`，月和日不足兩位要補 0，例如 `"2026-07-05"`。
- 如果格式打錯，網站部署時會自動失敗（不會把壞掉的資料上線），
  你在 GitHub 的 **Actions** 頁面可以看到哪裡出錯。
- 檔案裡本來的「範例：」那幾筆，等你熟悉後可以整段刪掉、換成你們自己的。
